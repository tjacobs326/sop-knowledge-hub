## Before You Begin

Use this template before opening a course build request or when another department needs to provide build details.

## Procedure

1. Copy the template into the approved intake location.
2. Add course code, course title, term, launch date, and program owner.
3. Identify whether the work is a new build, revision, copy, or remediation.
4. List required source files and where they are stored.
5. Add build instructions for modules, assessments, discussions, media, and rubrics.
6. List required external tools, links, or integrations.
7. Add accessibility notes, known exceptions, and deadline constraints.
8. Share the completed template with Course Operations.

## Template Fields

- Course code and title
- Term and launch date
- Request type
- Course owner and reviewer
- Source material location
- D2L tools required
- Accessibility notes
- Known issues
- Deadline and priority

## Troubleshooting / Notes

- If a field is unknown, mark it as pending and assign an owner.
- Attach the completed template to the course build request.
