## Before You Begin

Identify the target course, launch term, program owner, source materials, and any known constraints before creating the request.

## Procedure

1. Confirm the course code, section, term, and launch date.
2. Identify the course owner and the primary reviewer.
3. Gather source documents, templates, media, assessment files, and rubrics.
4. Confirm whether the build is new, revised, or copied from an existing shell.
5. List required D2L tools, integrations, and external links.
6. Document accessibility requirements or known content concerns.
7. Attach source materials to the request location.
8. Submit the course build request through the approved intake channel.
9. Monitor the request for follow-up questions from Course Operations.

## Troubleshooting / Notes

- Incomplete requests may be returned for clarification before work starts.
- Add immovable launch deadlines in the request title and priority notes.
- Use the Course Build Request Template when gathering information from another department.
