## Before You Begin

Confirm the course build is ready for final review, the course owner has completed content changes, and the QA checklist is available.

## Procedure

1. Open the course shell as a reviewer.
2. Confirm the course homepage, navigation, and key dates.
3. Review all modules for missing, duplicated, or outdated content.
4. Test links, embedded media, documents, and external tools.
5. Review assessments, gradebook categories, rubrics, and due dates.
6. Check headings, alt text, file names, and color contrast for accessibility.
7. Document issues with location, expected result, actual result, and screenshot.
8. Send issues to the assigned owner for correction.
9. Recheck resolved issues.
10. Mark the course approved for launch when all blocking findings are resolved.

## Troubleshooting / Notes

- Use a separate issue row for each finding so owners can track resolution.
- Do not approve launch when learner access, assessment configuration, or critical content is incomplete.
- If a build requires exception approval, attach the exception note to the QA record.
