# Accessibility Standards Baseline

The SOP Knowledge Hub should be built and reviewed against WCAG Level AA.

## Current Baseline

- ADA.gov's 2024 Title II web and mobile app rule identifies WCAG 2.1 Level AA as the technical requirement for covered state and local government web content and mobile apps.
- W3C WCAG 2.2 is the current W3C Recommendation and keeps the same core principles: perceivable, operable, understandable, and robust.
- For this project, use WCAG 2.1 AA as the minimum target and apply WCAG 2.2 AA-minded improvements where practical.

## Practical Review Checklist

- Page structure uses one clear H1 and does not skip heading levels.
- Interactive controls use native buttons, links, inputs, labels, and selects whenever possible.
- ARIA is used only when it improves semantics and uses valid roles, names, and states.
- Color contrast meets WCAG AA for normal text, small labels, badges, and button text.
- Color is never the only way status, priority, or meaning is communicated.
- Keyboard users can reach and operate all controls, with visible focus indicators.
- Forms have programmatic labels, clear required fields, and accessible error or success messages.
- Images and media have useful alternatives, captions, or labels when they convey information.
- Layout remains readable and operable at mobile, tablet, laptop, desktop, zoomed, and narrow-width views.
- Automated scans are run as a first pass, followed by manual keyboard and screen reader review.
